# datasets/

This directory **pins** the versioned dataset artifacts for the Pytheum public
release — `MANIFEST.json` records each artifact's name, sha256, size, and
consumer.

**Distribution note:** the `.jsonl.gz` artifacts themselves are **not committed
to this repo and not bundled in the PyPI wheel** — only their checksums are
pinned here. The hosted API (`https://api.pytheum.com`) serves the full current
graph with zero setup; to self-host with data, point
`PYTHEUM_EQUIVALENCE_PATH` / `PYTHEUM_RELATED_PATH` at export files. Without
them, dataset routes return empty results with a `file_missing` flag — never
errors.

## MANIFEST.json schema

Each entry in the `artifacts` array describes one released file:

```jsonc
{
  "name": "equivalence-export",        // logical dataset name
  "filename": "equivalence-export.jsonl.gz",
  "sha256": "<hex digest>",            // SHA-256 of the compressed file
  "records": 142179,                   // row count after decompression
  "description": "...",
  "license": "CC-BY-4.0",
  "version": "2026-06-13"
}
```

## Checksums

Each `.gz` artifact carries a `.sha256` sidecar generated by
`scripts/gen_checksums.py` and verified in CI by `scripts/verify_checksums.py`.

`../.gitattributes` carries a Git-LFS rule for `.gz` files so locally staged
artifacts never enter git history as blobs; as of now no artifact is pushed to
the repo — the manifest checksums are the source of truth for verifying a copy
obtained elsewhere (`scripts/verify_checksums.py`).
